## Тестирование создания объекта

### Описание

**Цель**: Проверка функциональности создания объекта в модуле «Объекты», включая проверку негативных сценариев.

Тестирование создания объекта с обязательными и необязательными полями, а также проверка обработки ошибок при неверном заполнении полей.

### Предусловия и необходимые инструменты

- Пользователь должен иметь доступ административной панели.
- Доступ администратора или суперадминистратора с разрешениями на просмотр объектов и создание объектов ("View Objects, Manage Objects").
- Доступ к изображениям для тестирования объектов.

### Тестовые данные
- Электронная почта: super@admin.com
- Пароль: [пароль из файла доступа в Notion]
- Наименование объекта: "Тестовый объект"
- Описание объекта: "Описание тестового объекта"
- Категория: "Тестовая категория"
- Вместимость: 100
- Чек-бокс "Активен": включен
- Главное изображение: test_image.jpg
- Галерея: gallery_image1.jpg, gallery_image2.jpg
- Теги: "Тестовый тег1", "Тестовый тег2"

### Шаги

#### Шаг 1:
Перейти на страницу создания объекта.
**Ожидаемый результат**: Страница создания объекта отображается корректно.

#### Шаг 2:
Заполнить обязательные поля: наименование объекта, описание объекта, категория, вместимость, активировать чек-бокс "Активен".
**Ожидаемый результат**: Поля принимают введенные данные.

#### Шаг 3:
Заполнить необязательные поля: главное изображение, галерея изображений, теги.
**Ожидаемый результат**: Поля принимают введенные данные.

#### Шаг 4:
Нажать кнопку "Создать".
**Ожидаемый результат**: Объект успешно создан и отображается в списке объектов.

### Негативные сценарии

#### Сценарий 1:
Не заполнять обязательные поля (наименование объекта, описание объекта, категория, вместимость).
**Ожидаемый результат**: Появляется сообщение об ошибке, указывающее на необходимость заполнения обязательных полей.

#### Сценарий 2:
Ввести некорректные данные в поле "Вместимость" (например, текст вместо числового значения).
**Ожидаемый результат**: Появляется сообщение об ошибке, указывающее на неправильный формат данных.

#### Сценарий 3:
Загрузить изображение неправильного формата (не изображение).
**Ожидаемый результат**: Появляется сообщение об ошибке, указывающее на неправильный формат изображения.

### Постусловия

- Удалить созданный тестовый объект (если он был создан).
- Восстановить первоначальные настройки, если были изменены.

### Замечания/Примечания

- Убедиться, что изображения загружаются корректно.
- Проверить, что чек-бокс "Активен" правильно сохраняет свое состояние.
