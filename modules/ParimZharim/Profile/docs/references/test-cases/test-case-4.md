## Получение профиля клиента через API

### Описание

**Цель**: Тестирование функции получения профиля клиента по токену через API.

Тестирование включает проверку валидности токена и соответствия данных профиля клиента записи в базе данных.

### Предусловия и необходимые инструменты

- Доступ к базе данных (таблица `profile_customers`).
- Инструмент для тестирования API, например, Postman.
- Валидный токен, полученный после выполнения test-case-1 или test-case-2.

### Тестовые данные

- Валидный токен: полученный после выполнения test-case-1 или test-case-2
- Невалидный токен: "invalidToken123"

### Шаги

#### Шаг 1: Получение профиля с валидным токеном
Отправить GET-запрос на адрес `/api/profile/customer/get-profile` с заголовком Authorization: Bearer {валидный токен}.
**Ожидаемый результат**: Код ответа 200. Возвращается модель пользователя, данные которой совпадают с записью в таблице `profile_customers`.

#### Шаг 2: Получение профиля с невалидным токеном
Отправить GET-запрос на адрес `/api/profile/customer/get-profile` с заголовком Authorization: Bearer invalidToken123.
**Ожидаемый результат**: Ошибка, код ответа 401 (Unauthorized), профиль не возвращается.

### Постусловия

- Удалить тестовые данные из таблицы `profile_customers` и `idm_users`, если были добавлены.
- Восстановить исходные настройки системы, если были изменены.

### Замечания/Примечания

- Необходимо убедиться, что коды ответов сервера и данные профиля соответствуют ожидаемым результатам.
- Проверить логирование событий для анализа возможных ошибок.
