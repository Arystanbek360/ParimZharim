## Тестирование Laravel Nova Admin панели

### Описание

**Цель**: Тестирование функционала Laravel Nova Admin панели для просмотра, создания, редактирования, сортировки, фильтрации и удаления данных клиентов.

Тестирование включает проверку возможности просмотра списка клиентов, просмотра карточки клиента, создания нового клиента, редактирования данных клиента, а также сортировки, фильтрации и удаления данных клиентов.

### Предусловия и необходимые инструменты

- Доступ к Laravel Nova Admin панели.
- Аккаунт супер-админа с e-mail super@admin.com и паролем, который хранится в Notion.
- Аккаунт администратора с правами просмотра и управления клиентами.

### Тестовые данные
- E-mail супер-админа: super@admin.com
- Пароль супер-админа: хранится в Notion
- Имя клиента: "Иван"
- Фамилия клиента: "Иванов"
- Телефон клиента: "+7 0000000777"
- Дата рождения клиента: "2000-01-01" (формат YYYY-MM-DD)
- Дубликат телефонного номера: "+7 0000000003" (для проверки ошибок при создании)

### Шаги

#### Шаг 1:
Авторизоваться в Laravel Nova Admin панели с использованием учетных данных супер-админа или администратора.
**Ожидаемый результат**: Пользователь успешно авторизуется и попадает на главную страницу панели.

#### Шаг 2:
Перейти в раздел «Профили клиентов».
**Ожидаемый результат**: Отображается список клиентов с их ID, фамилией, именем, телефоном и датой рождения.

#### Шаг 3:
Проверить возможность сортировки списка клиентов по имени, фамилии, телефону и дате рождения.
**Ожидаемый результат**: Список клиентов корректно сортируется по выбранному полю.

#### Шаг 4:
Проверить возможность фильтрации списка клиентов по дате рождения.
**Ожидаемый результат**: Список клиентов корректно фильтруется по указанной дате рождения.

#### Шаг 5:
Использовать поиск для нахождения клиента по имени или телефону.
**Ожидаемый результат**: Поиск успешно находит клиентов по введенному имени или телефону.

#### Шаг 6:
Нажать на клиента в списке для просмотра карточки клиента.
**Ожидаемый результат**: Отображается карточка клиента с полной информацией о нем и потенциально список его заказов.

#### Шаг 7:
Отредактировать данные клиента (имя, фамилию, телефон или дату рождения) в карточке клиента.
**Ожидаемый результат**: Данные клиента успешно сохраняются и отображаются обновленными.

#### Шаг 8:
Создать нового клиента, введя имя, фамилию, телефон и дату рождения.
**Ожидаемый результат**: Новый клиент успешно создается и отображается в списке клиентов.

#### Шаг 9:
Попробовать создать клиента с уже существующим номером телефона.
**Ожидаемый результат**: Отображается ошибка о том, что клиент с этим телефоном уже существует.

#### Шаг 10:
Удалить данные клиента, выбрав клиента галочкой на экране списка или перейдя в карточку клиента и выбрав действие «Удалить данные клиента».
**Ожидаемый результат**: Профиль клиента изменяется следующим образом: имя и фамилия сменяются на «Deleted User», номер телефона сменяется на "0000000000`$customerID`", дата рождения удаляется.

### Постусловия

- Очистить тестовые данные из системы при необходимости.
- Восстановить настройки панели при необходимости.

### Замечания/Примечания

- Тестирование предполагает наличие корректных учетных данных для доступа к панели.
- Для проверки отображения данных может потребоваться доступ к административным инструментам.
