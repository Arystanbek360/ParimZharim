## Регистрация нового клиента в API

### Описание

**Цель**: Тестирование функции регистрации нового клиента в API с использованием валидного и невалидного токена, проверки обязательных полей и валидации возраста.

Тестирование включает проверку регистрации нового клиента с обязательными полями (имя и дата рождения), валидацию возраста клиента (не менее 18 лет), и проверку наличия телефона, связанного с пользователем.
Данные отправляются через POST-запрос, а в ответе ожидается создание записи в таблице профилей клиентов и возвращение данных профиля.

### Предусловия и необходимые инструменты

- Доступ к API `/api/profile/customer/register` с возможностью отправки POST-запросов.
- Валидный токен, полученный в предыдущих тест-кейсах.
- Инструмент для отправки запросов, например Postman.

### Тестовые данные
- Валидный токен: valid_token (предполагается получение из test-case-1, test-case-2)
- Невалидный токен: invalid_token
- Имя: "Test User" (строка, максимум 255 символов)
- Дата рождения: "2000-01-01" (формат YYYY-MM-DD)
- Невалидная дата рождения: "2010-01-01" (меньше 18 лет)

### Шаги

#### Шаг 1:
Отправить POST-запрос с валидным токеном, именем и валидной датой рождения на URL.
**Ожидаемый результат**: Возвращается код ответа 200 и данные профиля клиента, включая customer_id, user_id, name, phone, email (чаще всего NULL), birth_date.

#### Шаг 2:
Отправить POST-запрос с валидным токеном и невалидной датой рождения (меньше 18 лет) на URL.
**Ожидаемый результат**: Возвращается код ошибки 422 с сообщением о валидации возраста.

#### Шаг 3:
Отправить POST-запрос с невалидным токеном на URL.
**Ожидаемый результат**: Возвращается код ошибки 401 с сообщением о неавторизованном доступе.

#### Шаг 4:
Проверить наличие записи в таблице профилей клиентов после успешного запроса.
**Ожидаемый результат**: В таблице `profile_customers` должна появиться новая запись с указанными полями.
В таблице `ìdm_users` должны обновиться данные.

### Постусловия

- Удалить тестовые данные из базы данных, если они были созданы.
- Восстановить настройки системы при необходимости.

### Замечания/Примечания

- Тестирование предполагает наличие валидных данных для токенов и корректной настройки системы.
- Для проверки записей в базе данных может потребоваться доступ к административным инструментам.
