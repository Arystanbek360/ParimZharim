## Проверка удаления данных пользователя в API

### Описание

**Цель**: Тестирование функции удаления данных пользователя в API с использованием валидного и невалидного токена и проверкой различных сценариев ошибок.

Тестирование включает проверку удаления данных пользователя с использованием DELETE-запроса, отправляемого на указанный URL.
В различных сценариях, включая отсутствие клиента, серверные ошибки и неправильный токен, ожидаются соответствующие коды ответов и сообщения об ошибках.

### Предусловия и необходимые инструменты

- Доступ к API с возможностью отправки DELETE-запроса на URL `/api/profile/customer/delete-profile`
- Валидный токен, полученный в test-case-1 и test-case-2.
- Инструмент для отправки запросов, например Postman.

### Тестовые данные
- Валидный токен: valid_token (предполагается получение из test-case-1 и test-case-2)
- Невалидный токен: invalid_token

### Шаги

#### Шаг 1:
Отправить DELETE-запрос с валидным токеном на `/api/profile/customer/delete-profile`
**Ожидаемый результат**: Возвращается код ответа 204, тело ответа пустое.

#### Шаг 2:
Отправить DELETE-запрос с валидным токеном для пользователя, у которого не найден клиент, на `/api/profile/customer/delete-profile`
**Ожидаемый результат**: Возвращается код ошибки 404 с сообщением «Customer not found».

#### Шаг 3:
Отправить DELETE-запрос с валидным токеном, имитируя серверную ошибку при удалении данных профиля, на `/api/profile/customer/delete-profile`
**Ожидаемый результат**: Возвращается код ошибки 500 с сообщением «Cannot delete customer profile with id: `$customerId`. Error: `$message`».

#### Шаг 4:
Отправить DELETE-запрос с невалидным токеном на `/api/profile/customer/delete-profile`.
**Ожидаемый результат**: Возвращается код ошибки 401 с сообщением о неавторизованном доступе.

#### Шаг 5:
Зайти в базу данных и проверить для этого кастомера, а также для юзера этого кастомера, что:
- Имя сменилось на "Deleted User".
- Номер телефона сменился на "0000000000`$customerID`".
- Email сменился на "deleted`$customerID`@deleted.user".
- Информация о дате рождения удалена.
  **Ожидаемый результат**: Все указанные поля изменены и информация удалена в соответствии с требованиями.


### Постусловия

- Удалить тестовые данные из базы данных, если они были созданы.
- Восстановить настройки системы при необходимости.

### Замечания/Примечания

- Тестирование предполагает наличие валидных данных для токенов и корректной настройки системы.
- Для проверки записей в базе данных может потребоваться доступ к административным инструментам.
