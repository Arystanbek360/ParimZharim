## Тестирование взаимодействия с профилями сотрудников в Laravel Nova Admin панели

### Описание

**Цель**: Тестирование функционала Laravel Nova Admin панели для просмотра, создания, редактирования, сортировки, фильтрации и удаления данных сотрудников.

Тестирование включает проверку возможности просмотра списка сотрудников, просмотра карточки сотрудника, создания нового сотрудника, редактирования данных сотрудника, а также сортировки, фильтрации и удаления данных сотрудников. Также проверяется создание и удаление пользователей, связанных с сотрудниками.

### Предусловия и необходимые инструменты

- Доступ к Laravel Nova Admin панели.
- Аккаунт супер-админа с e-mail super@admin.com и паролем, который хранится в Notion.
- Аккаунт администратора с правами просмотра и управления клиентами.

### Шаги

#### Шаг 1:
Авторизоваться в Laravel Nova Admin панели с использованием учетных данных супер-админа или администратора.
**Ожидаемый результат**: Пользователь успешно авторизуется и попадает на главную страницу панели.

#### Шаг 2:
Перейти в раздел «Профиль сотрудники».
**Ожидаемый результат**: Отображается список сотрудников с их ID, именем, фамилией, должностью, телефоном и e-mail.

#### Шаг 3:
Проверить возможность сортировки списка сотрудников по имени, фамилии, должности, телефону и e-mail.
**Ожидаемый результат**: Список сотрудников корректно сортируется по выбранному полю.

#### Шаг 4:
Проверить возможность фильтрации списка сотрудников по имени, фамилии или должности.
**Ожидаемый результат**: Список сотрудников корректно фильтруется по указанному полю.

#### Шаг 5:
Использовать поиск для нахождения сотрудника по имени, фамилии или должности.
**Ожидаемый результат**: Поиск успешно находит сотрудников по введенному имени, фамилии или должности.

#### Шаг 6:
Нажать на сотрудника в списке для просмотра карточки сотрудника.
**Ожидаемый результат**: Отображается карточка сотрудника с полной информацией о нем.

#### Шаг 7:
Отредактировать данные сотрудника (имя, фамилию, должность, телефон или e-mail) в карточке сотрудника.
**Ожидаемый результат**: Данные сотрудника успешно сохраняются и отображаются обновленными.

#### Шаг 8:
Создать нового сотрудника, введя имя, фамилию, должность, телефон и e-mail.
**Ожидаемый результат**: Новый сотрудник успешно создается и отображается в списке сотрудников. Проверить, что пользователь для этого сотрудника также создан.

#### Шаг 9:
Попробовать создать сотрудника с уже существующим e-mail.
**Ожидаемый результат**: Отображается ошибка о том, что сотрудник с этим e-mail уже существует.

#### Шаг 10:
Удалить данные сотрудника, нажав на значок корзины рядом с сотрудником в списке или в карточке сотрудника.
**Ожидаемый результат**: Сотрудник удаляется. Проверить, что пользователь этого сотрудника также удален.

### Постусловия

- Очистить тестовые данные из системы при необходимости.
- Восстановить настройки панели при необходимости.

### Замечания/Примечания

- Тестирование предполагает наличие корректных учетных данных для доступа к панели.
- Для проверки отображения данных может потребоваться доступ к административным инструментам.
