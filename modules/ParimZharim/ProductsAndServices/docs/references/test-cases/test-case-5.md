## Проверка создания категории меню

### Описание

**Цель**: Тестирование функции создания новой категории меню.

Проверка работы формы создания категории меню, включая все поля для ввода данных, чек-бокс, а также функциональность кнопок "Отмена", "Создать и добавить другой" и "Создать категория меню".

### Предусловия и необходимые инструменты

- Пользователь должен иметь доступ административной панели.
- Доступ администратора или суперадминистратора с разрешениями на просмотр и редактирование таблицы меню.
- Установленный веб-браузер с доступом к веб-приложению.

### Тестовые данные

- Электронная почта: super@admin.com
- Пароль: [пароль из файла доступа в Notion]
- Название категории: "Десерты"
- Видимость для клиента: Да

### Шаги

#### Шаг 1:
Перейти на страницу создания категории меню.
**Ожидаемый результат**: Страница создания категории меню отображается корректно, видны все поля для ввода данных и чек-бокс.

#### Шаг 2:
Заполнить поле "Название категории" значением "Десерты".
**Ожидаемый результат**: Поле "Название категории" принимает введенное значение.

#### Шаг 3:
Установить чек-бокс "Видимость для клиента".
**Ожидаемый результат**: Чек-бокс отмечен.

#### Шаг 4:
Нажать на кнопку "Отмена".
**Ожидаемый результат**: Создание категории меню отменено, пользователь возвращается на страницу таблицы категорий меню.

#### Шаг 5:
Заполнить все поля заново и нажать кнопку "Создать и добавить другой".
**Ожидаемый результат**: Категория меню сохранена, страница обновляется для создания новой категории меню.

#### Шаг 6:
Заполнить все поля заново и нажать кнопку "Создать категория меню".
**Ожидаемый результат**: Категория меню сохранена, пользователь перенаправлен на страницу просмотра созданного элемента меню.

### Постусловия

- Удалить созданные тестовые данные, если они были добавлены в ходе тестирования.
- Восстановить исходные настройки и данные в таблице категорий меню.
