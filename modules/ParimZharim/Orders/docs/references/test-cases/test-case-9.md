## Получение карточки объекта по ID через API

### Описание

**Цель**: Тестирование функции получения детальной информации об объекте по его идентификатору через API.

Проверка правильности вывода данных объекта с учетом возможности указания даты для получения актуальной информации на выбранный день.

### Предусловия и необходимые инструменты

- Доступ к API.
- Идентификатор существующего объекта.

### Тестовые данные

- Существующий объект ID: 2.
- Несуществующий объект ID: 404.

### Негативные кейсы

#### Шаг 1:
Отправить GET-запрос на `/api/orderable-service-objects/get` без указания `id`.
**Ожидаемый результат**: Возвращается статус-код 400 с сообщением об ошибке "Missing object ID".

#### Шаг 2:
Отправить GET-запрос на `/api/orderable-service-objects/get?id=404` для несуществующего объекта.
**Ожидаемый результат**: Возвращается статус-код 404 с сообщением об ошибке "Object not found".

### Примеры запросов и ответов для негативных кейсов

#### Отсутствие ID объекта:
GET `/api/orderable-service-objects/get`
**Ожидаемый результат**:
```json
{
    "message": "Не указан ID объекта",
    "status_code": 400
}
```

#### Несуществующий объект:
GET `/api/orderable-service-objects/get?id=404`
**Ожидаемый результат**:
```json
{
    "message": "Domain Error: Не найден объект 404",
    "status_code": 404
}
```

### Постусловия

- Убедиться, что данные об объекте корректно отображаются в системе при успешном запросе.
- Проверить, что система корректно обрабатывает отсутствие ID объекта и возвращает соответствующую ошибку.

### Замечания/Примечания

- Проверить корректность работы системы при обработке запросов на получение информации об объектах.
- Убедиться, что система корректно возвращает ошибки для несуществующих объектов и запросов без необходимых параметров.
