## Получение списка заказов через API

### Описание

**Цель**: Тестирование получения списка заказов через API для авторизованного клиента с заказами и без заказов.

Проверка процесса получения списка заказов, включая сортировку по дате и обработку ошибок для неавторизованных пользователей.

### Предусловия и необходимые инструменты

- Авторизованный клиент с профилем клиента и заказами.
- Авторизованный клиент с профилем клиента без заказов.
- Доступ к API по адресу `/api/orders/get-orders`.
- Токены авторизованных клиентов.

### Тестовые данные
- Токен клиента с заказами: [токен клиента с заказами].
- Токен клиента без заказов: [токен клиента без заказов].
- Неверный или отсутствующий токен: [неверный токен].

### Позитивные кейсы

#### Шаг 1:
Отправить GET-запрос на `/api/orders/get-orders` с токеном клиента с заказами.
**Ожидаемый результат**: Возвращается статус-код 200 и массив объектов заказов, отсортированный по дате (самые новые заказы вверху):
```json
[
  {
    "booking_date": "2024-06-09",
    "booking_id": 123,
    "object_name": "Название объекта",
    "object_category": "Категория объекта",
    "order_status": "Статус заказа",
    "stay_period": "20:00 - 02:00"
  },
  ...
]
```

#### Шаг 2:
Отправить GET-запрос на `/api/orders/get-orders` с токеном клиента без заказов.
**Ожидаемый результат**: Возвращается статус-код 200 и пустой массив:
```json
[]
```

### Негативные кейсы

#### Шаг 1:
Отправить GET-запрос на `/api/orders/get-orders` без токена или с неверным токеном.
**Ожидаемый результат**: Возвращается статус-код 401 с сообщением об ошибке "Unauthorized".

### Примеры запросов и ответов для негативных кейсов

#### Без токена или с неверным токеном:
GET `/api/orders/get-orders`
Headers:
```
Authorization: Bearer [неверный токен]
```
**Ожидаемый результат**:
```json
{
 "error": "Unauthorized"
}
```

### Постусловия

- Убедиться, что клиент может повторно авторизоваться при необходимости.
- Убедиться, что данные заказов корректно отображаются для клиентов с заказами.

### Замечания/Примечания

- Проверить корректность работы авторизации через токен.
- Убедиться, что заказы возвращаются отсортированными по дате, самые новые заказы вверху.
- Убедиться, что система корректно обрабатывает запросы от клиентов без заказов.
- Проверить, что неавторизованные пользователи получают ошибку 401.
