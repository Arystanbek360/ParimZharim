## Редактирование существующего заказа в админ-панели Nova Laravel

### Описание

**Цель**: Тестирование функции редактирования существующего заказа в админ-панели Nova Laravel.

Проверка корректности редактирования заказа, включая изменение времени, длительности, количества гостей, объекта и клиента, а также проверка валидации временных параметров и условий.

### Предусловия и необходимые инструменты

- Доступ к админ-панели Nova Laravel.
- Авторизация под пользователем с ролью superadmin (e-mail: super@admin.com) или любым другим пользователем с разрешением на управление и просмотр заказов.
- Наличие созданного объекта с добавленным расписанием и тарифным планом.
- Наличие существующего заказа для редактирования (статусы «Отменен» и «Выполнен» недоступны для редактирования).

### Тестовые данные
- Электронная почта: super@admin.com
- Пароль: [пароль из файла доступа в Notion]
- Объект с привязанным расписанием и тарифным планом.
- Существующий заказ для редактирования.
- Информация о клиенте для редактирования заказа.

### Шаги

#### Шаг 1:
Войти в систему с использованием учетных данных.
**Ожидаемый результат**: Успешный вход в систему.

#### Шаг 2:
Перейти в раздел просмотра всех заказов.
**Ожидаемый результат**: Отображается список заказов.

#### Шаг 3:
Выбрать существующий заказ, доступный для редактирования (не в статусах «Отменен» и «Выполнен»).
**Ожидаемый результат**: Открывается детальная карточка заказа для редактирования.

#### Шаг 4:
Изменить необходимые поля заказа (время, длительность, количество гостей, объект, клиент).
**Ожидаемый результат**: Поля заполняются корректно, вводимые данные принимаются.

#### Шаг 5:
Если заказ уже начат, попытаться изменить дату начала.
**Ожидаемый результат**: Дата начала недоступна для редактирования.

#### Шаг 6:
Сохранить изменения заказа.
**Ожидаемый результат**: Заказ сохраняется с пересчетом цены в соответствии с измененными параметрами.

#### Шаг 7:
Проверить валидацию временных параметров при изменении времени:
- Указана дата начала, которая позже даты окончания.
- У объекта нет расписания.
- У объекта нет тарифного плана.
- Длительность заказа слишком мала.
- Длительность заказа слишком велика.
- Время заказа занято другим заказом.
- Слот заказа находится в прошлом.
**Ожидаемый результат**: Система выполняет проверку и выдаёт соответствующие ошибки:
  - Ошибка, если дата начала позже даты окончания.
  - Ошибка, если у объекта нет расписания.
  - Ошибка, если у объекта нет тарифного плана.
  - Ошибка, если длительность заказа слишком мала.
  - Ошибка, если длительность заказа слишком велика.
  - Ошибка, если время заказа занято другим заказом.
  - Ошибка, если слот заказа находится в прошлом.

### Постусловия

- Выход из системы для очистки сессии.
