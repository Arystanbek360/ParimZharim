## Создание и редактирование тарифов и расписаний в админ-панели Novo Laravel

### Описание

**Цель**: Тестирование функции создания и редактирования тарифов и расписаний в админ-панели Novo Laravel.

Проверка корректности создания тарифов и расписаний, добавления правил для дней недели и конкретных дней, а также редактирования существующих тарифов и расписаний.

### Предусловия и необходимые инструменты

- Доступ к админ-панели Nova Laravel.
- Авторизация под пользователем с ролью superadmin (e-mail: super@admin.com) или любым другим пользователем с разрешением на управление и просмотр заказов.

### Тестовые данные
- Электронная почта: super@admin.com
- Пароль: [пароль из файла доступа в Notion]
- Название тарифа, базовый лимит гостей, стоимость за доп. гостя, правила для дней недели и конкретных дней.

### Шаги

#### Шаг 1:
Войти в систему с использованием учетных данных.
**Ожидаемый результат**: Успешный вход в систему.

#### Шаг 2:
Перейти в раздел «Настройки бронирования».
**Ожидаемый результат**: Открывается раздел «Настройки бронирования».

#### Шаг 3:
Нажать на кнопку «Создать тариф».
**Ожидаемый результат**: Открывается форма создания тарифа.

#### Шаг 4:
Ввести название тарифа, базовый лимит гостей, стоимость за доп. гостя и выбрать опцию.
**Ожидаемый результат**: Поля заполняются корректно.

#### Шаг 5:
Добавить правила для дней недели:
- Выбрать дни недели.
- Указать время с и до.
- Указать цену.
  **Ожидаемый результат**: Правила для дней недели добавлены корректно.

#### Шаг 6:
Добавить правила для конкретных дней:
- Выбрать конкретные дни.
- Указать время с и до.
- Указать цену.
  **Ожидаемый результат**: Правила для конкретных дней добавлены корректно.

#### Шаг 7:
Сохранить тариф.
**Ожидаемый результат**: Тариф успешно сохранен.

#### Шаг 8:
Перейти в редактирование созданного тарифа.
**Ожидаемый результат**: Открывается форма редактирования тарифа.

#### Шаг 9:
Убрать одно из правил для дней недели или добавить новые дни недели, обновить тариф.
**Ожидаемый результат**: Изменения в правилах для дней недели внесены корректно.

#### Шаг 10:
Сохранить изменения.
**Ожидаемый результат**: Тариф успешно обновлен и сохранен.

#### Шаг 11:
Проверить, что данные обновлены корректно.
**Ожидаемый результат**: Изменения в тарифе отображаются корректно.

#### Шаги по созданию и редактированию расписаний:

#### Шаг 12:
Перейти в раздел «Расписание».
**Ожидаемый результат**: Открывается раздел «Расписание».

#### Шаг 13:
Нажать кнопку «Создать расписание».
**Ожидаемый результат**: Открывается форма создания расписания.

#### Шаг 14:
Заполнить название расписания.
**Ожидаемый результат**: Название расписания заполнено корректно.

#### Шаг 15:
Добавить правила для дней недели:
- Выбрать дни недели.
- Указать время с и до.
- Указать цену.
  **Ожидаемый результат**: Правила для дней недели добавлены корректно.

#### Шаг 16:
Добавить правила для конкретных дней недели:
- Выбрать конкретные дни недели.
- Указать время с и до.
- Указать цену.
  **Ожидаемый результат**: Правила для конкретных дней добавлены корректно.

#### Шаг 17:
Сохранить расписание.
**Ожидаемый результат**: Расписание успешно сохранено.

#### Шаг 18:
Перейти в редактирование созданного расписания.
**Ожидаемый результат**: Открывается форма редактирования расписания.

#### Шаг 19:
Убрать одно из правил для дней недели или добавить новые дни недели, обновить расписание.
**Ожидаемый результат**: Изменения в правилах для дней недели внесены корректно.

#### Шаг 20:
Сохранить изменения.
**Ожидаемый результат**: Расписание успешно обновлено и сохранено.

#### Шаг 21:
Проверить, что данные обновлены корректно.
**Ожидаемый результат**: Изменения в расписании отображаются корректно.


### Постусловия

- Убедиться, что тариф и расписания созданы и обновлены корректно.

### Замечания/Примечания

- При создании тарифов и расписаний убедитесь, что все поля заполнены корректно и данные сохранены.
