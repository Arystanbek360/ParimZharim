## Получение списка объектов по ID категории и дате через API

### Описание

**Цель**: Тестирование API для получения списка объектов, доступных для заказа, на основе выбранной категории и даты.

Проверка доступности объектов в зависимости от заданных параметров и авторизации пользователя.

### Предусловия и необходимые инструменты

- Авторизованный клиент с доступом к API.
- Токен авторизации.

### Тестовые данные

- Токен авторизованного клиента: [токен клиента].
- Категория ID: 5 (пример существующей категории).
- Дата: "2024-07-20".

### Позитивные кейсы

#### Шаг 1:
Отправить GET-запрос на `/api/orderable-service-objects/get-by-category-id?category_id=5&date=2024-07-20` с токеном клиента.
**Ожидаемый результат**: Возвращается статус-код 200 и список объектов, соответствующих категории и дате, или пустой массив, если нет доступных объектов:

### Постусловия

- Убедиться, что клиент может повторно авторизоваться при необходимости.
- Проверить, что система корректно отображает доступные объекты или возвращает пустой массив при отсутствии доступных объектов на указанную дату.

### Замечания/Примечания

- Проверить корректность работы авторизации через токен.
- Убедиться, что система корректно обрабатывает и возвращает данные о доступных объектах в зависимости от выбранной категории и даты.
- В системе должна быть предусмотрена возможность корректно обрабатывать запросы на получение объектов по несуществующим или некорректным категориям, возвращая пустой массив в случае отсутствия соответствующих объектов.
