## Получение свободных слотов для объекта на выбранную дату через API

### Описание

**Цель**: Тестирование функциональности API для получения списка доступных временных слотов для объекта на конкретную дату.

Проверка доступности слотов с учетом авторизации пользователя и корректности введенных данных.

### Предусловия и необходимые инструменты

- Авторизованный клиент с доступом к API.
- Токен авторизации.

### Тестовые данные

- Токен авторизованного клиента: [токен клиента].
- Существующий объект ID: 3.
- Несуществующий объект ID: 400.
- Дата: "2024-12-20".

### Позитивные кейсы

#### Шаг 1:
Отправить GET-запрос на `/api/orderable-service-bject/get-free-slots-for-object-and-date?object_id=3&date=2024-12-20` с токеном клиента.
**Ожидаемый результат**: Возвращается статус-код 200 и массив свободных слотов для объекта на указанную дату:
```json
[
  {
    "start": "10:00",
    "min_duration": 1,
    "max_duration": 4
  },
  {
    "start": "14:00",
    "min_duration": 1,
    "max_duration": 4
  }
]
```

### Негативные кейсы

#### Шаг 1:
Отправить GET-запрос на `/api/orderable-service-bject/get-free-slots-for-object-and-date?date=2024-12-20` без указания `object_id`.
**Ожидаемый результат**: Возвращается статус-код 400 с сообщением об ошибке "Не указан ID объекта".

#### Шаг 2:
Отправить GET-запрос на `/api/orderable-service-bject/get-free-slots-for-object-and-date?object_id=400&date=2024-12-20` с токеном клиента.
**Ожидаемый результат**: Возвращается статус-код 404 с сообщением об ошибке "Object not found".

#### Шаг 3:
Отправить GET-запрос на `/api/orderable-service-bject/get-free-slots-for-object-and-date?object_id=3&date=2024-12-20` без токена или с неверным токеном.
**Ожидаемый результат**: Возвращается статус-код 401 с сообщением об ошибке "Unauthorized".

### Примеры запросов и ответов для негативных кейсов

#### Отсутствие ID объекта:
GET `/api/orderable-service-bject/get-free-slots-for-object-and-date?date=2024-12-20`
**Ожидаемый результат**:
```json
{
    "message": "Не указан ID объекта",
    "status_code": 400
}
```

#### Объект не найден:
GET `/api/orderable-service-bject/get-free-slots-for-object-and-date?object_id=400&date=2024-12-20`
**Ожидаемый результат**:
```json
{
    "message": "Domain Error: Не найден объект 400",
    "status_code": 404
}
```

#### Неавторизованный доступ:
GET `/api/orderable-service-bject/get-free-slots-for-object-and-date?object_id=3&date=2024-12-20`
Headers:
```
Authorization: Bearer [неверный токен]
```
**Ожидаемый результат**:
```json
{
  "error": "Unauthorized"
}
```

### Постусловия

- Убедиться, что клиент может повторно авторизоваться при необходимости.
- Проверить, что система корректно отображает доступные слоты или возвращает соответствующие ошибки при отсутствии данных или неправильном доступе.

### Замечания/Примечания

- Проверить корректность работы авторизации через токен.
- Убедиться, что система корректно обрабатывает и возвращает данные о доступных временных слотах в зависимости от выбранного объекта и даты.
- Важно проверить, что система возвращает соответствующие ошибки при некорректных запросах или отсутствии данных.
