## Отмена заказа в админ-панели Novo Laravel

### Описание

**Цель**: Тестирование функции отмены заказа в админ-панели Nova Laravel.

Проверка корректности отмены заказа, отображения кнопки отмены для подходящих статусов заказа и недоступности редактирования заказа после отмены.

### Предусловия и необходимые инструменты

- Доступ к админ-панели Nova Laravel.
- Авторизация под пользователем с ролью superadmin (e-mail: super@admin.com) или любым другим пользователем с разрешением на управление и просмотр заказов.
- Наличие существующего заказа в статусах «Создан», «Подтвержден» или «Запрошена отмена».

### Тестовые данные
- Электронная почта: super@admin.com
- Пароль: [пароль из файла доступа в Notion]
- Существующий заказ в одном из допустимых статусов.

### Шаги

#### Шаг 1:
Войти в систему с использованием учетных данных.
**Ожидаемый результат**: Успешный вход в систему.

#### Шаг 2:
Перейти к заказу в одном из допустимых статусов через список заказов или через лензу «Запрос отмены».
**Ожидаемый результат**: Открывается список заказов или детальная карточка заказа.

#### Шаг 3:
Убедиться, что кнопка «Отменить заказ» отображается только для заказов в статусах «Создан», «Подтвержден» или «Запрошена отмена».
**Ожидаемый результат**: Кнопка «Отменить заказ» отображается только для подходящих статусов заказа.

#### Шаг 4:
Нажать кнопку «Отменить заказ».
**Ожидаемый результат**: Заказ успешно отменяется, статус заказа меняется на «Отменен».

#### Шаг 5:
Проверить, что заказ больше не доступен для редактирования.
**Ожидаемый результат**: Заказ недоступен для редактирования, доступна только опция копирования.

### Постусловия

- Заказ корректно помечен как «Отменен», недоступен для редактирования.

### Замечания/Примечания

- Заказ, находящийся в статусе «Создан», автоматически отменяется по истечению времени подтверждения.
