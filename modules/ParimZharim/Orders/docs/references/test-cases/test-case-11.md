
## Просмотр списка заказов в Laravel Nova Admin Panel

### Описание

**Цель**: Тестирование возможности просмотра и фильтрации списка заказов в панели администратора Laravel Nova.

Тестирование функционала просмотра, сортировки, фильтрации и действий для списка заказов в панели администратора Laravel Nova.

### Предусловия и необходимые инструменты

- Доступ администратора или суперадминистратора с разрешениями на просмотр объектов ("View Orders").
- Наличие объектов в разных статусах для отображения в различных разделах.

### Тестовые данные
- Электронная почта: super@admin.com
- Пароль: [пароль из файла доступа в Notion]
- Заказы с различными статусами:
  - Статус: "Отменен"
  - Статус: "Завершен"
  - Статус: "Подтвержден"
  - Статус: "Начат"
  - Статус: "Создан"
  - Статус: "Запрос на отмену"

### Шаги

#### Шаг 1: 
Перейти на страницу "Все заказы".
**Ожидаемый результат**: Отображается список всех заказов, отсортированных по времени начала (самые новые сверху).

#### Шаг 2: 
Проверить возможность сортировки списка по различным полям (ID, категория, название объекта, клиент, время начала, время окончания, статус).
**Ожидаемый результат**: Список корректно сортируется по выбранному полю.

#### Шаг 3: 
Использовать мегафильтр для фильтрации заказов по категории объекта, статусу заказа, дате заказа, создавшему администратору.
**Ожидаемый результат**: Отображаются заказы, соответствующие установленным фильтрам.

#### Шаг 4: 
Использовать поле поиска для поиска заказов по клиенту или комментариям к заказу.
**Ожидаемый результат**: Отображаются заказы, соответствующие введенным критериям поиска.

#### Шаг 5: 
Перейти к объекту с статусом "Отменен" и проверить наличие трех точек с действием "Создать копию".
**Ожидаемый результат**: Для отмененного заказа доступно действие "Создать копию".

#### Шаг 6: 
Перейти к объекту со статусом "Не завершен" и проверить наличие возможности редактирования.
**Ожидаемый результат**: Заказ доступен для редактирования.

#### Шаг 7: 
Проверить доступность действия выгрузки в Excel для страниц всех заказов.
**Ожидаемый результат**: Доступна выгрузка всех полей заказов в Excel.

#### Шаг 8: 
Перейти на страницу "Активные заказы" и убедиться, что отображаются заказы со статусами "Подтвержден" и "Начат" за текущий день.
**Ожидаемый результат**: Отображаются соответствующие заказы с возможностью фильтрации.

#### Шаг 9: 
Перейти на страницу "Ожидает оплаты" и убедиться, что отображаются заказы со статусом "Создан".
**Ожидаемый результат**: Отображаются заказы со статусом "Создан" с доступными фильтрами по категории объекта, администратору и дате заказа.

#### Шаг 10: 
Перейти на страницу "Внести в АЙКО" и убедиться, что отображаются заказы, требующие синхронизации с АЙКО-системой.
**Ожидаемый результат**: Отображаются заказы с фильтрами по категории объекта, администратору и дате заказа.

#### Шаг 11: 
Перейти на страницу "Прием к объектам" и убедиться, что отображаются заказы со статусами "Время" и "Стекло".
**Ожидаемый результат**: Отображаются соответствующие заказы с фильтрами по категории объекта и администратору, с возможностью подтвердить выполнение заказа.

#### Шаг 12: 
Перейти на страницу "Запрос отмены" и убедиться, что отображаются заказы с запросом на отмену.
**Ожидаемый результат**: Отображаются заказы со статусом "Запрос на отмену".

### Постусловия

- Восстановление первоначальных фильтров и сортировок.
- Удаление тестовых данных, созданных в ходе тестирования, если они не нужны для дальнейшей работы.

### Замечания/Примечания

Возможность выгрузки в Excel позволяет экспортировать все поля заказов, даже те, которые не отображаются в списке.
