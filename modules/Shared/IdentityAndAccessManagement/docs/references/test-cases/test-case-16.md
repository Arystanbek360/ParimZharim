## Проверка работы разрешений согласно их описаниям

### Описание

**Цель**: Проверка правильности работы каждого разрешения в системе путем создания тестового пользователя и проверки всех разрешений в соответствии с их описаниями.

Тестировщик должен создать тестового пользователя, прикрепить к нему роли с соответствующими разрешениями, и затем проверить, что действия, разрешенные данным пользователям, соответствуют описаниям разрешений.

### Предусловия и необходимые инструменты

- Пользователь должен иметь доступ к админ-панели.
- Пользователь должен обладать правами на создание и редактирование пользователей, ролей и разрешений.
- Необходимо наличие активной сессии пользователя.
- Доступ к документации или описаниям разрешений для проверки их корректности.

### Тестовые данные
- Тестовый пользователь: test.user@example.com
- Роли: TestRole1, TestRole2 (роли, содержащие разные разрешения)
- Разрешения: ViewReports, EditUsers, DeleteRecords (перечень разрешений, которые нужно проверить)

### Шаги

#### Шаг 1:
Зайти в админ-панель.
**Ожидаемый результат**: Пользователь успешно авторизуется в админ-панели.

#### Шаг 2:
Создать нового тестового пользователя с логином `test.user@example.com`.
**Ожидаемый результат**: Тестовый пользователь успешно создан.

#### Шаг 3:
Создать роли `TestRole1` и `TestRole2` с различными наборами разрешений
**Ожидаемый результат**: Роли с соответствующими разрешениями успешно созданы.

#### Шаг 4:
Прикрепить тестовому пользователю роли `TestRole1` и `TestRole2`.
**Ожидаемый результат**: Роли успешно прикреплены к тестовому пользователю.

#### Шаг 5:
Зайти в систему под тестовым пользователем и проверить доступ к действиям, связанным с выбранными разрешениями
**Ожидаемый результат**: Доступные действия должны соответствовать описаниям разрешений

#### Шаг 6:
Снять одно из разрешений, и повторно зайти в систему под тестовым пользователем.
**Ожидаемый результат**: Пользователь больше не должен иметь доступ к действиям, связанным со снятым разрешением

### Негативные сценарии

#### Негативный сценарий 1:
Попробовать выполнить действие, на которое у пользователя нет разрешения.
**Ожидаемый результат**: Пользователь не имеет доступа к разделу, для которого у пользователя нет разрешений

#### Негативный сценарий 2:
Попробовать дать тестовому пользователю противоречащие друг другу разрешения (например, разрешение на создание, но не на редактирование объектов).
**Ожидаемый результат**: Проверить, как система обрабатывает конфликтующие разрешения. Ожидается, что система корректно обрабатывает такие случаи и применяет наиболее строгие правила.

### Постусловия

- Удалить тестового пользователя `test.user@example.com`.
- Удалить созданные роли и связанные с ними разрешения, чтобы вернуть систему в исходное состояние.

### Замечания/Примечания

- Документация по разрешениям должна быть актуальной, чтобы тестирование соответствовало текущей конфигурации системы.
