
## Проверка консольной команды по удалению старых записей в базе данных

### Описание

**Цель**: Тестирование консольной команды, которая раз в сутки удаляет записи старше двух месяцев из таблицы idm_phone_verification_codes.

Проверка корректности выполнения команды по удалению старых записей.

### Предусловия и необходимые инструменты

- Доступ к базе данных.
- Доступ к запуску консольной команды в кубере.

### Тестовые данные
- Таблица idm_phone_verification_codes с записями старше и младше двух месяцев.

### Шаги

#### Шаг 1:
Зафиксировать запись в таблице idm_phone_verification_codes с created_at больше двух месяцев назад.
**Ожидаемый результат**: Запись успешно зафиксирована в базе данных.

#### Шаг 2:
Зафиксировать запись в таблице idm_phone_verification_codes с created_at менее двух месяцев назад.
**Ожидаемый результат**: Запись успешно зафиксирована в базе данных.

#### Шаг 3:
Запустить консольную команду по удалению старых записей.
**Ожидаемый результат**: Команда успешно выполняется.

#### Шаг 4:
Проверить записи в таблице idm_phone_verification_codes после выполнения команды.
**Ожидаемый результат**: Записи с CreatedAt больше двух месяцев назад удалены. Записи с created_at менее двух месяцев назад остаются в базе данных.

### Постусловия

- Убедиться, что оставшиеся записи в базе данных актуальны и корректны.

### Замечания/Примечания

- Убедиться, что консольная команда запускается корректно и удаляет только те записи, которые старше двух месяцев.
- Опционально: Проверить выполнение команды в 12 ночи по времени сервера.
