
## Авторизация и разлогин в админ панели Laravel Nova

### Описание

**Цель**: Тестирование сценария авторизации и разлогина в админ панели Laravel Nova.

Проверка процесса входа в админ панель Nova с использованием корректных и некорректных учетных данных, а также процесса выхода из учетной записи.

### Предусловия и необходимые инструменты

- Доступ к админ панели Laravel Nova по адресу host/nova/login.
- Тестовые учетные данные пользователей с ролью ADMIN или SUPER_ADMIN.

### Тестовые данные
- Электронная почта:  from DB for user with  ADMIN or SUPER_ADMIN roles
- Пароль: from DB for user with  ADMIN or SUPER_ADMIN roles
- Пароль: incorrectPassword (неверный пароль)

### Позитивные кейсы

#### Шаг 1:
Перейти на страницу авторизации по адресу host/nova/login.
**Ожидаемый результат**: Отображается форма авторизации с полями для ввода электронной почты и пароля, и кнопкой «Войти».

#### Шаг 2:
Ввести корректную электронную почту и пароль.
**Ожидаемый результат**: Поля для ввода электронной почты и пароля принимают введенные данные.

#### Шаг 3:
Нажать кнопку «Войти».
**Ожидаемый результат**: Пользователь авторизуется и попадает на главную страницу дашборда.

#### Шаг 4:
Проверить отображение данных на дашборде.
**Ожидаемый результат**: Пользователь видит данные согласно своей ролевой модели.

#### Шаг 5:
Нажать на имя пользователя в правом верхнем углу.
**Ожидаемый результат**: Отображается выпадающий список с опцией «Выйти».

#### Шаг 6:
Нажать на кнопку «Выйти» и подтвердить выход.
**Ожидаемый результат**: Пользователь выходит из системы и перенаправляется на экран авторизации.

### Негативные кейсы

#### Шаг 1:
Перейти на страницу авторизации по адресу host/nova/login.
**Ожидаемый результат**: Отображается форма авторизации с полями для ввода электронной почты и пароля, и кнопкой «Войти».

#### Шаг 2:
Ввести корректную электронную почту и некорректный пароль.
**Ожидаемый результат**: Поля для ввода электронной почты и пароля принимают введенные данные.

#### Шаг 3:
Нажать кнопку «Войти».
**Ожидаемый результат**: Отображается ошибка «Указанные учетные данные не соответствуют записи».

### Постусловия

- Пользователь не авторизован в системе, сессия очищена.
- Убедиться, что пользователь может повторно авторизоваться при необходимости.

### Замечания/Примечания

- Убедиться, что данные сессии пользователя полностью удалены после разлогина.
- Проверить корректное отображение стартового экрана после выхода из аккаунта.
