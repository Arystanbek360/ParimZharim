
## Проверка работы роутинга в административной панели Laravel Nova

### Описание

**Цель**: Тестирование работы роутинга в административной панели Laravel Nova.

Проверка редиректов и отображения правильных страниц в зависимости от состояния авторизации и существования маршрутов.

### Предусловия и необходимые инструменты

- Доступ к админ панели Laravel Nova по адресу host/nova/login.

### Тестовые данные
- Неавторизованный пользователь

### Шаги

#### Шаг 1:
Перейти по существующей ссылке на ресурс в административной панели для неавторизованного пользователя.
**Ожидаемый результат**: Отображается экран логина.

#### Шаг 2:
Перейти по адресу host напрямую.
**Ожидаемый результат**: Отображается экран логина.

#### Шаг 3:
Перейти по адресу host/12345. Где 12345 - несуществующий маршрут.
**Ожидаемый результат**: Отображается страница ошибки 404.

### Постусловия

- Неавторизованный пользователь остается неавторизованным.

### Замечания/Примечания

- Убедиться, что редиректы и отображение страниц работают корректно для неавторизованных пользователей.
- Проверить, что страница ошибки 404 отображается корректно для несуществующих маршрутов.
