
## Тестирование просмотра изменения контента с помощью API

### Описание

**Цель**: Тестирование отображения изменений контента через API-запрос.

Тестирование процесса редактирования контента на странице в модуле CMS и последующего просмотра изменений с помощью GET-запроса через клиент (например, Postman).

### Предусловия и необходимые инструменты

- Доступ к модулю CMS в системе.
- Наличие учетной записи с правами на редактирование контента.
- Установленный клиент для отправки API-запросов (Postman или аналогичный).

### Тестовые данные
- Пример страницы для редактирования: {{host}}/privacy-policy

### Шаги

#### Шаг 1: 
Перейти на страницу модуля CMS, в раздел "Контент".
**Ожидаемый результат**: Страница модуля CMS "Контент" загружена корректно.

#### Шаг 2: 
Выбрать любую страницу с текущим созданным контентом, например, {{host}}/privacy-policy".
**Ожидаемый результат**: Открывается страница с текущим контентом.

#### Шаг 3: 
Нажать кнопку "Редактировать" и внести изменения в контент страницы.
**Ожидаемый результат**: Поля для ввода данных позволяют редактировать текст.

#### Шаг 4: 
Нажать кнопку "Обновить" для сохранения изменений.
**Ожидаемый результат**: Изменения успешно сохранены, отображается сообщение об успешном обновлении.

#### Шаг 5: 
Открыть клиент для отправки API-запросов (например, Postman).
**Ожидаемый результат**: Клиент для отправки API-запросов открыт и готов к использованию.

#### Шаг 6: 
Отправить GET-запрос к API с использованием слага, например:
```
GET {{host}}/api/cms/get-content-by-slug?slug=privacy-policy
```
**Ожидаемый результат**: В ответе API отображается обновленный контент страницы.

#### Шаг 7:
Отправить GET-запрос к API с использованием несуществующего слага, например:
```GET /api/cms/get-content-by-slug?slug=123```
**Ожидаемый результат**: API возвращает ошибку 404 (Not Found) и сообщение:
```
{
    "message": "Content not found",
    "status_code": 404
}
```

### Постусловия

- Проверить, что изменения на странице соответствуют внесенным в модуле CMS и отображаются корректно через API-запрос.
- В случае необходимости восстановить первоначальный контент страницы.

### Замечания/Примечания

- Убедитесь, что клиент для API-запросов настроен правильно и имеет доступ к нужному API.
- Проверьте, что обновленный контент отображается корректно через API на всех устройствах и браузерах.
