# Сервисы для модуля "Документы"

## Введение

Слой сервисов в модуле "Документы" предоставляет централизованное управление всеми операциями, связанными с документами.
Сервисы отвечают за логику создания, обновления, получения и архивации документов, а также за управление доступом.

## Сервисы

### DocumentService

#### Основные функции:

- `createNewDocument(Document)`: Создание нового документа.
- `updateDocument(Document)`: Обновление документа с созданием новой версии.
- `validateAccessToDocument(Document, User)`: Проверка прав доступа пользователя к документу.

### PackageService

#### Основные функции:

- `saveDocumentInPackage(Document, Package)`: Сохранение документа в пакете документов.
- `createNewPackage(Package)`: Создание нового пакета документов.
- `updatePackage(Package)`: Обновление информации о пакете документов.
- `validateAccessToPackage(Package, User)`: Проверка прав доступа пользователя к пакету документов.

## Расширенная функциональность

Эти сервисы также поддерживают различные операции запросов и фильтрации для удобства работы пользователей и
администраторов системы.

### QueryService

#### Основные функции:

- `getDocument(id)`: Получение документа по идентификатору.
- `getDocumentsByQuery(query)`: Поиск документов по различным критериям (имя, номер, тип и т.д.).
- `getDocumentHistory(DocumentID)`: Получение истории версий документа.
- `getDocumentByIDs(ids)`: Получение документов по списку идентификаторов.
- `getDocumentsByTypeAndDate(type, date_from_from, date_from_to)`: Получение документов по типу и датам.
- `getPackage(id)`: Получение пакета по идентификатору.
- `getPackagesByQuery(query)`: Поиск пакетов по различным критериям (имя, тип и т.д.).
