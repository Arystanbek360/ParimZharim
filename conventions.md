## Git
Для работы с git-ветками используем Gitflow:

- `develop` - ветка с актуальной версией кода
- feature/ - ветки для разработки новых функций
- hotfix/ - ветки для исправления ошибок в стабильной версии

Правила именования в feature/ и hotfix/ ветках:
- `feature/YYYY-MM-DD-название-функции` - для новых функций
- `hotfix/YYYY-MM-DD-название-исправления` - для исправления ошибок

В рамках ветки делается pull request, он направляется в `develop`. После проверки кода и успешного прохождения тестов ветка сливается в `develop`. После этого ветка удаляется.
